Recommended Remediation Command — STIG Alignment Criteria

The STIG alignment joins observed appliance evidence and STIG-required evidence on the same measurable coordinate. For each measurable where the comparison fails, the STIG alignment classifies the mismatch into one of three alignment actions based on the shape of the evidence:

ELIMINATE — required is absent, observed is present
The STIG criteria require the measurable to be zero, empty, false, or absent — but the appliance has a non-empty value. The observed object, account, binding, or setting must be removed or disabled.
STIG pattern: the criteria expression tests for == 0, == false, or == true (where true means “restricted”).
V-266065 — auth_user_shared_accounts == 0 → shared/group user accounts exist and must be deleted.
V-266066 — auth_user_local_account_count == 1 → extra local accounts exist beyond the single emergency account and must be removed.
V-266084 — virtual_server_services_restricted == true → virtual servers listening on disallowed ports must be disabled.
ADD — required is present, observed is absent
The STIG criteria require a non-empty, non-zero value — but the appliance observation is empty, null, or missing. The required server, setting, policy item, or binding must be created or attached.
STIG pattern: the criteria expression tests for >= N where N > 0, or for a required value that is not yet present.
V-266075 — sys_syslog_remote_server_count >= 2 → no remote syslog servers configured; at least two must be added.
V-266076 — sys_ntp_server_count >= 2 → no NTP servers configured; at least two must be added.
V-266079 — auth_mfa_configured == 'tacacs' → no authentication source configured; TACACS+ must be set up.
UPDATE — both present, values differ
Both the STIG and the appliance have a value for the same measurable, but they do not match. The existing setting must be changed in place to equal the STIG-required value.
STIG pattern: the criteria expression tests for a specific value or threshold against the same measurable.
V-266064 — sys_httpd_max_clients <= 10 → observed value exceeds 10; must be changed to 10.
V-266069 — auth_password_policy_max_login_failures <= 3 → current failure limit too high; must be changed to 3.
V-266068 — sys_daemon_log_tmm_os_log_level == 'informational' → log level set to wrong value; must be changed to “informational”.

Remediation Guidance

This control requires manual remediation. The guidance above describes the steps needed.
Method:

Recommended TMSH Remediation Command

TMSH remediation translates the STIG alignment action into a shell command from the contract tmsh_equivalent field. The TMSH verb maps directly to the alignment class:

ELIMINATE → tmsh delete or tmsh modify ... disabled
V-266065: tmsh delete auth user <shared_account>; tmsh save sys config
V-266066: tmsh delete auth user <extra_account>; tmsh save sys config
ADD → tmsh modify ... add or tmsh modify ... enabled
V-266075: tmsh modify sys syslog remote-servers add { stig_syslog1 { host <ip1> ... } stig_syslog2 { host <ip2> ... } }; tmsh save sys config
V-266076: tmsh modify sys ntp servers add { <ntp_server1> <ntp_server2> }; tmsh save sys config
UPDATE → tmsh modify with the STIG-required value
V-266064: tmsh modify sys httpd max-clients 10; tmsh save sys config
V-266069: tmsh modify auth password-policy max-login-failures 3; tmsh modify auth password-policy lockout-duration 900; tmsh save sys config

Recommended REST Remediation Command

REST (iControl) remediation translates the STIG alignment action into an HTTP operation from the contract endpoint and payload fields. The HTTP method maps to the alignment class:

ELIMINATE → manual DELETE or PATCH to disabled state (PATCH_COLLECTION with selector)
V-266065: PATCH_COLLECTION /mgmt/tm/auth/user with selector description_contains_any: ["shared","group","generic","service"] — manual user deletion.
V-266084: PATCH /mgmt/tm/ltm/virtual/<name> {"disabled": true} for disallowed-port listeners.
ADD → PATCH with payload that populates an empty collection or enables a missing setting
V-266075: PATCH /mgmt/tm/sys/syslog {"remoteServers": [{"name":"stig_syslog1","host":"<ip1>"},{"name":"stig_syslog2","host":"<ip2>"}]}
V-266076: PATCH /mgmt/tm/sys/ntp {"servers": ["<ntp_server1>","<ntp_server2>"]}
UPDATE → PATCH with the STIG-required value for an existing resource
V-266064: PATCH /mgmt/tm/sys/httpd {"maxClients": 10}
V-266069: PATCH /mgmt/tm/auth/password-policy {"maxLoginFailures": 3, "lockoutDuration": 900}
